Docker VS Code Remote Container
使用Remote Container建置專案開發環境
6月 5, 2022

環境檢查

安裝：

  • vs code
  • docker

Step 1

安裝Remte Development

開啟VS Code => 安裝Extension “Remote Development”


Remote Development裡包含了Remote-Containers，Remote-SSH，Remote-WSL

Step 2

設定 Container

安裝完成後嘗試使用的話可以選擇Try a Development Container Sample

這裡直接開始使用，首先我目前開發的專案是Angular + Dotnet Core 6 ， 因此我的開發環境會需要以下：

  • dotnet
  • node js
  • npm
  • angular-cli

Step 1

開啟你要想Remote Container的專案後，選擇Add Development Container Configuration Files

下一步選擇你要的環境，這裡我選Dotnet

後面會問你要選擇的版本，照你的需求選即可！

Step 2

選擇完成以後，會在你的專案產生一個資料夾.devcontainer

這裡我們看Dockerfile

註解上都有詳細說明作用了，這裡因為我是使用Dotnet Core 6，因此我將版本改成6.0

Step 3

確認好Dockerfile以後，我們選擇“Reopen in Container”

會幫我們把Project開在Container裡面，要確認可以從TERMINAL或標題確認

確認好在Container後，輸入

如果都有回覆目前版本，代表Container內的環境設定有成功

再來安裝Angular

  • dotnet
  • node js
  • npm
  • angular-cli

完成以後，照平常專案的編譯方式，即可正常開發使用

透過這樣的方式建置開發環境，除了讓本機開發環境乾淨以外，如果電腦突然壞掉了也不用過於難受，Container開起來就可以繼續用不同的電腦開發了，同理如果有其他人要一起開發，也可以透過這個方式，讓對方能迅速進入開發，真棒啊～
